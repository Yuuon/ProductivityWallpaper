# SVG图片转化XAML UI 说明文件

根据AGENTS.md中对项目的说明与AGENTS_UI_Workflow.md中对UI处理流程的规范，按需处理以下提及的UI的修改或新建。
所有生成的UI默认语言使用英文词条Key，并补充对应的英文和中文本地化翻译

* MainWindow
参考文件："ReferenceResources/我的主题 - 使用记录 - 无记录.svg";
注意：该SVG文件包含其他视图元素，但是相对来说最为简洁，故使用此图做参考；MainWindow应该只包含窗体和上方导航栏；导航栏最左端为Logo加应用标题，中间为标签页按钮，点击切换不同的视图，选中状态时文字使用主题色，右侧为设置，最小化，关闭图标。图标文件位于Resources\Img下。

* WorkshopView (新建)
参考文件："ReferenceResources/page_workshop_default.svg" - 默认，"ReferenceResources/创意工坊 - 有搜索结果.svg" - 搜索结果；
该视图主要为链接steam API,显示创意工坊中的相关内容。具体的功能将在后期进行开发，目前UI只进行占位处理；
视图分为左右两块区域。左边为主显示区，分为上下两层，上层为搜索/排序栏，从左到右依次为搜索条（支持文字搜索与tag搜索），分辨率下拉菜单，与排序下拉菜单（热度，最新等）。下层为具体的动态内容显示，带垂直滚动条。
右边为详情区域，当左边动态内容中有项目被选中时显示该项目相关信息，自上到下分别是：预览图，主题名字，上传用户的名字，文件大小/分辨率，主题标签，立即使用按钮，去修改按钮。
注意图标示例中标签有一个高亮，其他非高亮，这只是演示高亮效果。默认所有标签应该是非高亮，当点击某标签时，高亮该标签，并在搜索栏启用该标签的标签搜索，支持复数标签选中。
当前使用中的主题，去使用按钮显示“使用中”。

* WallpaperView (重构为MyThemeView)
参考文件："ReferenceResources/page-my-theme.svg" - 有具体内容时，"ReferenceResources/我的主题 - 使用记录 - 无记录.svg" - 无具体内容时；
该视图主要显示本地文件，结构与WorkshpView基本一致，唯一的区别是搜索/排序栏改为“使用记录”和“我的作品”两个按钮，分别显示从创意工坊缓存到本地和用户自己创作的主题内容。当这两个页面没有内容时，在中央位置显示nocontents图标和提示语“这里空空如也\n去找找你喜欢的主题吧~”（我的作品把“找找”改成“创作”），然后下方是跳转按钮“去瞅瞅”，点击跳转Workshop页面（同理“我的作品”按钮显示“去创作”，点击跳转创作主题页面）。

* CreatorView (新建)
参考文件："ReferenceResources/创作主题 - 空.svg" - 进入创作主题的首页，"ReferenceResources/创作主题 - 主题预览 - 空.svg" - 开始创作主题的第一步页面；
创作主题页面首页在中央显示欢迎词：“把桌面，变成你喜欢的样子”，同时下方文本输入框，占位词显示“输入主题名字...”，然后再正文为主题色按钮“创作主题”，点击后进入创作主题第一步；
进入第一步后，页面分为左右两部分：左侧为一列按钮，每一项为具体的创作功能，点击时进入相应页面并高亮按钮（主题色），具体功能如下：“主题预览”，“桌面背景(x)”，“鼠标点击(x)”，“关机(x)”，“开机重启(x)”，“屏幕唤醒(x)”，“打开应用”，“桌面时钟”，“番茄专注钟”，“纪念日”，其中有些项目中的(x)为当前项目已经配置的内容数量，比如桌面背景配置了几张壁纸就显示几。
右侧为创作内容具体的操作区，进入第一步时，默认显示主题预览内容，在新创作时，内容为空，区域内显示内容如下：上方一行主题名称，显示首页输入的名字，后方追加笔图标按钮，点击可以再修改主题名称；下面分左右两列，左边是主题整体画面预览，右边是配置项，在没有内容时，画面预览中央显示提示词“这里空空如也\n去创作你的专属主题吧~”。

由于创作主题功能较为复杂，暂时先新建两个页面，确保主页的功能跳转能够正确实现。
